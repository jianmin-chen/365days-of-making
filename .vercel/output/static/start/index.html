How to start one

Start by forking the source and then cloning your fork and installing the necessary dependencies:

git clone https://github.com/<your username>/365days-of-making
cd 365days-of-making
npm i  # or yarn install

We’ll also need to set up Firebase so you can get people subbed to your updates. Head over to Firebase, log into the appropriate account, and click on “Go to console” in the top right corner. Click on the big plus button that shows up, which will take you through creating a Firebase project.

Once you have one created, we’re going to create a web app in our Firebase project. Click on the code icon that will show up and follow the instructions to create a web app. Make sure you save the Firebase configuration, it’ll look like this:

const firebaseConfig = {
  apiKey: "",
  authDomain: "",
  projectId: "",
  storageBucket: "",
  messagingSenderId: "",
  appId: "",
  measurementId: ""
}

This will go inside /src/firebase/client.ts. Replace the current firebaseConfig with yours.

That’s what the client will be able to use (backed up by secure rules). Since we’re the admin, we’d like to have full access to everything, though. Let’s set up admin access by grabbing our project credentials. You can generate them by going under your project settings (which will be a settings icon next to your project name in the sidebar) > Service accounts > Firebase admin SDK > Generate new private key.

To add these, we’ll add them to our list of environment variables. Create a file named .env in the root directory, and add these values in:

FIREBASE_PRIVATE_KEY_ID=YOUR_PRIVATE_KEY_ID
FIREBASE_PRIVATE_KEY=YOUR_PRIVATE_KEY
FIREBASE_PROJECT_ID=YOUR_PROJECT_ID
FIREBASE_CLIENT_EMAIL=YOUR_CLIENT_EMAIL
FIREBASE_CLIENT_ID=YOUR_CLIENT_ID
FIREBASE_AUTH_URI=YOUR_AUTH_URI
FIREBASE_TOKEN_URI=YOUR_TOKEN_URI
FIREBASE_AUTH_CERT_URL=YOUR_AUTH_CERT_URL
FIREBASE_CLIENT_CERT_URL=YOUR_CLIENT_CERT_URL

Move setup.ts into /pages/api/auth. We’re going to set up an admin account with your personal email (or as many as you’d like, so make sure to use the appropriate email. Or you could create multiple logins. Whatever you’d like.)

Now we’re going to start Astro:

npm run dev

Head over to the link Astro gives you and you should now see this website! Wooh.

To get ourselves an admin account, let’s now download Postman. Once you get Postman downloaded, you’ll see something like this:

Create a new request with the Add button and then set it to be POST instead of the default GET. Go over to the Body tab and select form-data, and add these two key-value pairs:

email=YOUR EMAIL
password=YOUR PASSWORD

If everything works out, you now have an account! Delete the setup.ts file and head to /signin, and you should be able to sign in, after which you will get redirected to the dashboard.

Let’s do one last thing for setup - set a start date. Head back to .env, and add in a line like the following:

PUBLIC_START_DATE=2023-09-15

(Prefixing our environment variables with a PUBLIC_ lets Astro know that a variable should be public.) This will be used on the main page to map all 365 days.

Now that you’ve got everything set up, let’s write our first entry. This can be about anything! For me, I introduced myself, what this was about, why I was doing it, and what I was going to be doing. You can check it out here.

Also feel free to edit this page you’re reading right now - that’s in /src/content/content/start.mdx, although it should stay relatively the same.

Setting up Giscus

To get comments on your 365 days of making, set up Giscus by going to https://giscus.app/ and selecting your preferred options. You’ll get a script that looks something like this:

<script
  src="https://giscus.app/client.js"
  data-repo="jianmin-chen/365days-of-making"
  data-repo-id="R_kgDOKRAyPw"
  data-category="Announcements"
  data-category-id="DIC_kwDOKRAyP84CZLWE"
  data-mapping="pathname"
  data-strict="0"
  data-reactions-enabled="0"
  data-emit-metadata="0"
  data-input-position="top"
  data-theme="light"
  data-lang="en"
  data-loading="lazy"
  crossorigin="anonymous"
  async></script>

You will need to replace this script at the bottom of src/pages/start.astro and src/pages/entry/[slug].astro. Once you do feel free to add a “Hello, world!” to your first entry to make sure it works!

Deploying

Let’s deploy. Set up an GitHub repository, and push the contents of your folder to the repository. Once you’re done, head over to Vercel, sign up if you haven’t, and create a new project. Link that project to your repository, and paste your environment variables where it asks you to.

Now head over to the link Vercel gives you and you’ll see your site in action!

Adding yourself to the webring

Please add yourself to the webring! I personally would love to check out other people’s 365 days of making. Head over here and open an issue with your name and a link to your website, and you’re all good.

Frequently asked questions

What if I want to do more than 365 days, or less?

Yeah! Add an variable to .env:

PUBLIC_NUMBER_OF_DAYS=365

What’s up with all this extra content in the dashboard?

I left that there for myself! To get rid of it, head over to /src/components/Editor.tsx and get rid of content={…}.

Is there an RSS feed I can subscribe to?

Yep! You can find it here.
